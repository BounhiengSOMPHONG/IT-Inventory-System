IT INVENTORY SYSTEM - API DOCUMENTATION
=====================================

1. DEPARTMENT ENDPOINTS
----------------------
CRUD operations for departments

GET /api/departments
- Get all departments
- Query params: search (optional)
- Example: GET /api/departments

POST /api/departments
- Create a new department
- Request body: {"name": "New Department Name"}
- Example: POST /api/departments {"name": "Finance Department"}

GET /api/departments/:id
- Get a specific department
- Example: GET /api/departments/1

PUT /api/departments/:id
- Update a department
- Requires: {"name": "Updated Department Name"}
- Example: PUT /api/departments/1 {"name": "Updated Finance Department"}

DELETE /api/departments/:id
- Delete a department
- Example: DELETE /api/departments/1

4. PRODUCT ENDPOINTS
--------------------
CRUD operations for products with Active/Inactive status

GET /api/products
- Get all products
- Query params: search, status (Active/Inactive)
- Example: GET /api/products?status=Active
- Example: GET /api/products?search=Dell

POST /api/products
- Create a new product
- Request body: {productName, productModel, manufacturer, productTypeId, assetCode, serialNumber, serviceTag, hd, ram, cpu, status, addedBy, yearBought}
- Example: 
  POST /api/products
  {
    "productName": "Dell Laptop",
    "productModel": "XPS 13",
    "manufacturer": "Dell",
    "productTypeId": 1,
    "assetCode": "IT005",
    "serialNumber": "DL123456",
    "status": "Inactive",
    "addedBy": 1,
    "yearBought": "2023"
  }

GET /api/products/inactive
- Get only inactive products (for creating service records)
- Example: GET /api/products/inactive

GET /api/products/deleted
- Get deleted products (admin only)
- Example: GET /api/products/deleted

GET /api/products/:id
- Get a specific product
- Example: GET /api/products/1

GET /api/products/:id/edit-logs
- Get edit logs for a specific product (admin only)
- Example: GET /api/products/1/edit-logs

PUT /api/products/:id
- Update a product
- Example: PUT /api/products/1 {"productName": "Updated Product Name"}

DELETE /api/products/:id
- Delete a product
- Example: DELETE /api/products/1

PATCH /api/products/:id/restore
- Restore a deleted product (admin only)
- Example: PATCH /api/products/1/restore

GET /api/products/report
- Export products report to Excel
- Query params: status (Active/Inactive)
- Example: GET /api/products/report?status=Active
- Example: GET /api/products/report (all products)

GET /api/products/change-report
- Export product change logs to Excel (admin only)
- Query params: search, productId
- Example: GET /api/products/change-report

6. SERVICE ENDPOINTS
--------------------
CRUD operations for services

GET /api/services
- Get all services
- Query params: statusId, employeeId
- Example: GET /api/services?statusId=1
- Example: GET /api/services?employeeId=1

POST /api/services
- Create a new service
- Request body: {employeeId, assetCode, date, statusId, serviceBy, remark}
- Example:
  POST /api/services
  {
    "employeeId": 1,
    "assetCode": "IT001",
    "date": "2023-10-15",
    "statusId": 1,
    "serviceBy": 1,
    "remark": "Assigned to employee"
  }

GET /api/services/:id
- Get a specific service
- Example: GET /api/services/1

PUT /api/services/:id
- Update a service
- Example: PUT /api/services/1 {"statusId": 2}

DELETE /api/services/:id
- Delete a service
- Example: DELETE /api/services/1

POST /api/services/assign
- Assign a product to an employee
- Request body: {assetCode, employeeId, remark}
- Example:
  POST /api/services/assign
  {
    "assetCode": "IT002",
    "employeeId": 2,
    "remark": "New laptop assignment"
  }

POST /api/services/return
- Return a product to stock
- Request body: {assetCode, remark}
- Example:
  POST /api/services/return
  {
    "assetCode": "IT002",
    "remark": "Returned by employee"
  }

GET /api/services/report
- Export services report to Excel
- Query params: statusId
- Example: GET /api/services/report?statusId=1
- Example: GET /api/services/report (all services)

GET /api/services/change-report
- Export service change report to Excel
- Example: GET /api/services/change-report

8. AUTHENTICATION ENDPOINTS
---------------------------
Authentication and authorization

POST /api/auth/register
- Register a new user
- Request body: {username, password, image, role}
- Example:
  POST /api/auth/register
  {
    "username": "testuser",
    "password": "testpassword",
    "role": "User"
  }

POST /api/auth/login
- Login user and get JWT token
- Request body: {username, password}
- Example:
  POST /api/auth/login
  {
    "username": "testuser",
    "password": "testpassword"
  }

POST /api/auth/change-password
- Change user password (auth required)
- Request body: {currentPassword, newPassword}
- Example:
  POST /api/auth/change-password (with auth header)
  {
    "currentPassword": "oldpassword",
    "newPassword": "newpassword"
  }

AUTHENTICATION
--------------
Most endpoints require authentication using JWT tokens.
Include in request header: Authorization: Bearer {token}

Admin endpoints require admin role:
- GET /api/users
- GET /api/products/deleted
- GET /api/products/:id/edit-logs
- PATCH /api/products/:id/restore
- DELETE /api/users/:id
- PUT /api/products/:id (uses auth info)

ERROR RESPONSES
---------------
Standard error format:
{
  "error": "Error message"
}

Success responses typically return:
{
  "message": "Success message",
  "data": { ... }
}
